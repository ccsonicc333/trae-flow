# TRAE FLOW 0.1.9

## 修复

- Mineradio 播放状态陈旧回退：WKWebView 不在视图层级（Flow 岛收起）时 JS 事件暂停，超过 3 秒无 playback 消息时用 MediaRemote 的 elapsed/duration 回退，避免 UI 卡在旧值或 0 秒
- Mineradio 查询代数过滤：每次 trackChanged 发起新查询时递增 generation，旧回调返回时若 generation 已过期则丢弃，防止旧歌数据覆盖新状态
- Mineradio trackChanged 立即用 pendingSongId 获取歌词，不再等待异步查询结果，切歌时歌词加载更及时
- Mineradio 切歌时清空 title/artist/coverURL，避免新歌无歌词时回退显示旧歌标题
- Mineradio 查询脚本只扫描 mineradio.art 明确的 `playQueue` 变量，不再扫描通用变量名（list/songs/queue 等），避免误匹配页面推荐/搜索/导航数组

## 说明

- 本次发版主要优化 Mineradio 在 Flow 岛收起场景下的播放状态稳定性与切歌时数据一致性
